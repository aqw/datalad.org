What is DataLad?

DataLad is an open source utility that brings the core concepts of version control systems to data â€” allowing versioning, branching, publishing, and merging with ease throughout the data lifecycle.

Leveraging git-annex and taking git's concept of decentralization a step further, DataLad's "data handles" enable the selective acquisition of files and, when configured, across mirrors and a variety of transport methods (HTTP, FTP, rsync, BitTorrent, and more).

The DataLad project also operates a crawler that regularly indexes datasets from scientific data portals such as OpenFMRI and CRCNS, making them trivial to acquire and work with using datalad.

Getting Started

DataLad covers all aspects of working with data: discovery, consumption, curation of original or derived data, and publication. The following pages illustrate how DataLad aids anyone working with data.

Looking for data?

DataLad has built-in support for metadata extraction and search. With only a few steps you can search through a large collection of readily available datasets, and immediately download them. See how this works...

Improve the reproducibility of your science?

DataLad provides joint management of analysis code and data. This enables you to comprehensively track the exact state of any analysis inputs that produced your results — across the entire lifetime of a project, and across multiple datasets. See how this works...

Need fine-grained data access?

DataLad offers easy access to individual files — great when you only need a few files from a large dataset for a web-demo or an analysis. Files in a dataset can be distributed across multiple download sources with tailored permissions to match your data privacy needs. See how this works...

Want to share your work?

DataLad supports sharing datasets with the public or just some colleagues on the platforms that you are using already — no need for a central service. You have complete freedom to share your work in multiple locations simultaneously (your own server, DropBox, GitHub, etc.) without loosing track. See how this works...
